# 开题报告草稿

## 一、 选题意义 (Significance of the Topic)

**1. 顺应鸿蒙生态发展需求，填补应用服务保障空缺**
随着 HarmonyOS NEXT（纯血鸿蒙）的发布与推广，构建丰富且高质量的原生应用生态成为当前的关键任务。在官方应用尚未完全覆盖或功能受限的过渡期，第三方应用（如本项目所服务的网易云音乐第三方播放器）在满足用户娱乐需求方面扮演着重要角色。然而，第三方应用往往依赖于非官方 API 接口，其稳定性直接决定了用户体验。本项目构建的 API 监测平台，正是为了解决这一核心痛点，为鸿蒙生态下的第三方应用提供坚实的后台服务保障。

**2. 解决第三方接口不稳定性痛点，提升用户体验**
第三方音乐播放器高度依赖后端 API 服务的可用性。由于网络波动、接口地址变更或反爬策略限制，API 服务可能出现间歇性中断或高延迟，导致用户出现“无法播放”、“加载失败”等问题。本系统通过建立一套可视化的 API 监测与状态发布机制，能够实时监控各服务节点的健康状况。对于用户而言，能够透明地了解服务状态，区分是本地网络问题还是服务端故障；对于开发者而言，能够第一时间发现并定位接口异常，从而快速切换备用节点或进行维护，显著提升应用的整体可靠性。

**3. 探索前后端分离与全栈监控技术的实践应用**
本项目不仅仅是一个简单的状态展示页面，而是一个集成了实时监控、数据统计、即时通知与后台管理的全栈系统。选题涉及 Vue 3 前端框架、Node.js 后端服务、WebSocket 实时通信、SQLite 轻量级数据库以及 ECharts 数据可视化等多项主流技术的综合应用。通过该项目的开发，能够深入探索如何构建高可用、低延迟的监控系统，验证跨平台（Web 端监控 + 鸿蒙端应用）服务架构的可行性，具有较高的工程实践价值。

## 二、 写作提纲 (Writing Outline)

**第一章 绪论**
1.1 **研究背景**
   - 鸿蒙 NEXT 操作系统生态现状及第三方应用的发展机遇。
   - 第三方音乐播放器在数据接口稳定性方面面临的挑战。
1.2 **研究意义**
   - 提升第三方应用服务质量与用户满意度。
   - 为开发者提供可视化的运维监控辅助工具。
1.3 **国内外研究现状**
   - 现有的服务监控平台（如 Uptime Robot 等）的优缺点分析。
   - 针对特定业务（音乐 API）的定制化监控方案缺失。
1.4 **本文主要工作**
   - 构建 API 监测与分发平台。
   - 实现多维度的服务状态可视化与后台管理功能。

**第二章 相关技术与工具**
2.1 **前端开发技术**
   - Vue 3 框架及其组合式 API（Composition API）。
   - Element Plus 组件库与响应式布局设计。
   - ECharts 数据可视化库在监控图表中的应用。
2.2 **后端开发技术**
   - Node.js 环境与 Express 框架。
   - SQLite 嵌入式数据库的设计与使用。
   - JWT（JSON Web Token）身份认证机制。
2.3 **实时通信技术**
   - WebSocket 协议在状态实时推送中的应用。
2.4 **开发环境与部署**
   - Vite 构建工具与 Git 版本控制。

**第三章 系统需求分析**
3.1 **功能性需求**
   - **实时监控功能**：对音乐 API 节点进行定时心跳检测，记录响应时间与可用率。
   - **数据可视化功能**：展示服务可用性热力图（UptimeHeatmap）、响应时间折线图。
   - **信息发布功能**：系统公告、应用版本更新与下载链接管理。
   - **后台管理功能**：管理员登录、API 节点增删改查、访客数据统计。
3.2 **非功能性需求**
   - **实时性**：状态变更需毫秒级同步至客户端。
   - **稳定性**：监控服务需 7x24 小时稳定运行。
   - **兼容性**：适配移动端与桌面端不同尺寸屏幕（服务于鸿蒙手机/平板用户）。

**第四章 系统设计**
4.1 **系统总体架构**
   - B/S 架构设计（浏览器/服务器模式）。
   - 前后端分离的数据交互流程。
4.2 **数据库设计**
   - E-R 图设计。
   - 数据表结构设计（访客表 `visitors`、音乐接口表 `music_apis`、系统日志表等）。
4.3 **功能模块设计**
   - 监控模块：定时任务调度与状态判定逻辑。
   - 统计模块：访客地理位置与设备类型分析算法。
   - 安全模块：登录鉴权与接口代理（解决 HTTPS 混合内容问题）。

**第五章 系统实现**
5.1 **前端页面实现**
   - 首页仪表盘（Dashboard）布局与交互。
   - 音乐 API 详情页与 Mini 播放器组件实现。
   - 后台管理系统（Admin）的路由守卫与动态菜单。
5.2 **后端服务实现**
   - RESTful API 接口开发（`/api/music-stats`, `/api/login` 等）。
   - 监控定时器与 WebSocket 广播机制代码实现。
   - 异常处理与日志记录。
5.3 **特色功能实现**
   - API 自动择优与连接持久化。
   - 深色模式（Dark Mode）的自适应适配。

**第六章 系统测试**
6.1 **测试环境与工具**
6.2 **功能测试**
   - 监控准确性测试。
   - 管理员权限控制测试。
6.3 **性能测试**
   - 高并发下的 WebSocket 推送延迟测试。
   - 页面加载速度优化与验证。
6.4 **测试结果分析**

**第七章 总结与展望**
7.1 **全文总结**
7.2 **系统不足与改进方向**
   - 分布式监控节点的引入。
   - 更多维度的鸿蒙设备适配优化。
